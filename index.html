Why LYS?

Train Game 2015 is the culmination of many months work from team LYS, team XJG and team FVS. It is an exciting turn based game that pits players against one another in order to fulfil different contracts with trains.

Build Connections, Create Rivalries

Not every station in the map is connected, so use your cash to build new connections in order to fulfil your goals. Using an adversaries connection means you will have to pay them rent, so plan your route carefully to maximise your payoff!

Protect yourself!

You can buy connections that are made out of Gold, Silver or Bronze.
Gold connections are the most expensive connections to buy and repair, but they do not take any damage from passing trains.
The health of your connection affect the speed of passing trains, so keep your connections in tip-top shape to get your trains further, faster! Connections can even be upgraded from one material to another!

Complete Control

The Heads Up Display always provides you with the information you need to make tactical decisions. Hover over a train to see where it is going, Select multiple stations to see if a connection is viable. Check the health of your connections in a pinch.

Stay on your toes

You have contracts to fulfil, some of which need to be completed in a limited number of trains. Watch out for junction failures, and collisions with other trains. A mishap could send you back to the drawing board!

Replays

Just annihilated your friend? Want to re-live the moment? You can playback the events of you last played game. If your friend can't bare to re-live his demise, you can even adjust the speed of the replay!

Code Quality

We have taken great care in re-factoring , and adding features to our inherited codebase, making it easy to read and extend. Using LibGDX, a very intuitive and powerful game framework that does the heavy lifting for you, means you can focus on implementing cool new features! It uses Gradle, for dependency management, helping your team organise any external libraries you wish to use.

The Code

Team LYS have worked very hard to ensure that our code is idiomatic, and easy to read. Don't just take our word for it though, have a look! Our repo is available on GitHub. 



    public void play() {
        if (!anyMoreEventsToPlay()) return;

        playBackReplayEvent(eventInstanceToPlayNext);
        eventInstanceLastPlayed = eventInstanceToPlayNext;

        int nextEventIndex = eventInstances.indexOf(eventInstanceLastPlayed) + 1;
        if (nextEventIndex < eventInstances.size()) {
            /*
            If there is an event to play, set up the timer to wait for the time
            specified by it's appropriate gameEvent, after which time the event
            will be fired.
             */
            eventInstanceToPlayNext = eventInstances.get(nextEventIndex);
            timer.scheduleTask(new com.badlogic.gdx.utils.Timer.Task() {
                @Override
                public void run() {
                    play();
                }
            }, eventInstanceToPlayNext.gameEvent.delay / playBackSpeed);
        } else {
            eventInstanceToPlayNext = null;
        }
    }
